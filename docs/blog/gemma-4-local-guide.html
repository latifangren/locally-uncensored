Google Gemma 4 — Run It Locally

Published April 10, 2026 · PurpleDoubleD · 6 min read

Google released Gemma 4 in April 2026 and it immediately became one of the most important open-weight model families for local AI. From the tiny E2B that runs on anything to the frontier 31B dense model, Gemma 4 covers every use case. Native tool calling, vision support, 256K context, and an Apache 2.0 license make it the go-to recommendation for new users.

This guide covers every Gemma 4 variant, how to run them locally, and which uncensored versions exist.

Why Gemma 4 Matters

Native Tool Calling

Gemma 4 supports structured tool calling out of the box. In Locally Uncensored, this means Agent Mode works without any workarounds. The model can search the web, read and write files, execute code, and interact with your system using the 13 built-in MCP tools.

Vision

Attach images to your messages and Gemma 4 analyzes them. Describe what is in a photo, extract text from screenshots, explain diagrams. This works across all Gemma 4 sizes except E2B.

Apache 2.0 License

Unlike Meta's Llama models (which have usage restrictions above 700M monthly users) or Qwen (which requires attribution), Gemma 4 uses Apache 2.0. You can use it for anything, commercial or personal, with no restrictions.

256K Context Window

The 27B model supports 256K tokens of context. That is roughly 500 pages of text in a single conversation. For the Coding Agent, this means analyzing entire codebases without losing context.

Uncensored Variants

The official Gemma 4 models have content filters. Google trained them to refuse certain prompts. The community has created uncensored versions:

Heretic Variants

The "heretic" uncensored method produces some of the cleanest unrestricted outputs. Available for 31B dense and 26B MoE. These keep full model quality while removing refusals.

Abliterated Variants

Abliteration removes the refusal direction from the model weights. Available for 27B. Slightly more aggressive uncensoring than heretic.

Which Gemma 4 Should You Pick?

4 GB VRAM or less: Gemma 4 E4B

The E4B is remarkably good for its size. It handles general chat, basic coding, and simple tool calling. It is the default recommended model in Locally Uncensored's onboarding for users with smaller GPUs.

16 GB VRAM: Gemma 4 31B

The full 31B dense model is the recommendation. Frontier-level quality with native vision and tool calling. For uncensored use, get the heretic variant.

Speed over quality: Gemma 4 26B MoE

The MoE variant has 26B total parameters but only activates 4B per token. It runs noticeably faster than the 31B dense model while keeping most of the quality. Good for interactive agent sessions where response time matters.
